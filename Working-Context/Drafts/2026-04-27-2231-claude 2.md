OpenClaw TTS config lives at `messages.tts` (global) or `channels.discord.tts` (Discord override). Two practical paths for Netcup:

**Option A: espeak-ng** (free, robotic, 30s setup)

```bash
# On Netcup as root
apt install -y espeak-ng

# As openclaw user
sudo -u openclaw bash -c 'export PATH=/home/openclaw/.npm-global/bin:$PATH && \
  openclaw config set messages.tts.auto always && \
  openclaw config set messages.tts.provider tts-local-cli && \
  openclaw config set messages.tts.providers.tts-local-cli.command espeak-ng && \
  openclaw config set messages.tts.providers.tts-local-cli.args '"'"'["-w", "{{OutputPath}}", "{{Text}}"]'"'"' && \
  openclaw config set messages.tts.providers.tts-local-cli.outputFormat wav'
```

Templates: `{{Text}}`, `{{OutputPath}}`, `{{OutputDir}}`, `{{OutputBase}}`. If `{{Text}}` omitted from args, text pipes to stdin instead.

**Option B: OpenAI TTS** (best quality, paid)

```bash
sudo -u openclaw bash -c 'export PATH=/home/openclaw/.npm-global/bin:$PATH && \
  openclaw config set messages.tts.auto always && \
  openclaw config set messages.tts.provider openai && \
  openclaw config set messages.tts.providers.openai.voice alloy'
```

Needs `OPENAI_API_KEY` in env or secrets.

**Discord voice-specific override:**

```bash
openclaw config set channels.discord.tts.auto always
openclaw config set channels.discord.tts.provider tts-local-cli
```

**Restart gateway:**

```bash
pkill -f openclaw-gateway
```

**How it works:**
- Discord: streams audio into voice channel directly
- Telegram: sends voice-note file
- `auto` modes: `always` (every reply), `inbound` (only when user sends voice), `tagged` (only when user types `/tts` or similar), `off`
- `mode`: `final` (TTS only final reply) vs `all` (TTS tool results too)

**Piper** (middle ground - neural, free, CPU) if you want better than espeak without API costs. Download binary + `.onnx` voice model, then configure same `tts-local-cli` pattern with `--stdin` and pipe text (omit `{{Text}}` from args).
