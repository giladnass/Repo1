OpenClaw uses stdio bridge via `mcp-remote`. Same pattern works for Codex CLI.

**Install bridge:**
```bash
npm install -g mcp-remote
```

**Codex config** (`~/.codex/config.json`):
```json
{
  "mcpServers": {
    "basic-memory": {
      "command": "mcp-remote",
      "args": [
        "https://memory.giladn.com/mcp/ebc69cbd338019004dfd1738a033ced1e22e21ff62a41b41ef0b8caf5d9fb3a5"
      ]
    }
  }
}
```

**Verify:**
```bash
codex --mcp-list
```

**Note:** This is read+write access. Same auth token in URL as OpenClaw and Perplexity use. If Codex project is public/shared, consider moving token to env var and using shell wrapper instead.
