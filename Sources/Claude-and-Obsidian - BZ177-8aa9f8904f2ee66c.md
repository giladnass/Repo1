# Obsidian-Claude Integration Cheat Sheet

## Summary

The document outlines how Obsidian, a note-taking app with plain-text markdown files, integrates with Claude Cowork and Claude Code to solve AI workflow challenges like session-based memory loss, compaction, disorganized folder structures, inconsistent prompts, and tool-switching friction. By storing AI-generated content in a persistent vault, users gain control over context, enable compounding knowledge, and streamline workflows through session logs, structured folder systems, process files, and custom commands. The system shifts AI usage from ad-hoc tasks to a scalable, evolving infrastructure.

## Key Findings

- Obsidian acts as a persistent memory layer for Claude, preserving session data across projects and preventing context loss due to AI compaction.
- Session logs in Obsidian create structured, searchable records that rebuild context after compaction events, reducing recovery time from hours to minutes.
- Intentional folder structures (e.g., PARA method) improve long-term maintainability, while CLAUDE.md files centralize global workflow preferences.
- Process files and slash commands automate repetitive workflows, turning ad-hoc prompts into versioned, improvable systems.
- Wikilinks and graph views in Obsidian enhance knowledge connectivity, enabling AI to map relationships between ideas and projects.
- The integration allows compounding value: every session adds to a personal knowledge base that future AI models can leverage more effectively.
